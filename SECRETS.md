# Расшифровка репозитория (git-crypt)

Папка [`docs/`](docs) в этом репозитории зашифрована через **git-crypt**.
Чтобы получить к ней доступ, нужно один раз разлочить репозиторий ключом.

> Файл этой инструкции лежит в корне (а не в `docs/`) намеренно: `docs/`
> зашифрована, и внутри неё инструкция была бы нечитаемой до разлочки.

## 1. Установить инструменты

```bash
brew install git-crypt openssl
```

(`git-crypt` — для расшифровки файлов, `openssl` — для расшифровки самого ключа.)

## 2. Разлочить репозиторий

Ключ git-crypt хранится в репозитории в зашифрованном виде:
`secrets/git-crypt.key.enc`. Пароль к нему передаётся отдельно (не в git).

```bash
cd secrets

# 1) расшифровать ключ git-crypt (спросит пароль)
openssl enc -d -aes-256-cbc -pbkdf2 -in git-crypt.key.enc -out git-crypt.key

# 2) разлочить репозиторий этим ключом
git-crypt unlock git-crypt.key

# 3) удалить расшифрованный ключ — он больше не нужен и не должен попасть в git
rm git-crypt.key
```

После `git-crypt unlock` файлы в `docs/` автоматически расшифруются в рабочей
копии, и с ними можно работать как обычно.

## Важно

- **`secrets/git-crypt.key`** (сырой ключ) — в `.gitignore`, его нельзя коммитить.
- В git попадает только **`secrets/git-crypt.key.enc`** (защищён паролем openssl).
- Пароль от openssl держится вне репозитория (менеджер паролей / передача лично).
